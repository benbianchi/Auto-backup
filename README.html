Project: Automatic File Sharing and Backup

    Table of Contents

  • Purpose
  • client.js
  • server.js
  • Conclusion

  • Purpose

    To create a two processed system that allows a system to automatically sync up rapidly editted files between multiple machines.
    A server must be set up with a user's configurations.

  • client.js

    Client.js starts with a shebang line (for us unix lovers).

    A globals section is set up for easy configuration. As of now, "auth" and "path" are unused.
    The process makes a request for a specified server, asking for files to download. If there are no files to be uploaded, then we recieve a 404 error. As files are downloaded they are compiled and saved to the client's disk.

  • server.js

    Once again, server.js starts with a shebang file.
    A dependencies and globals section has been added for fast configuration. If you have not installed the dependecies required, you will be unable to run the script. The server does generate a log for the user to increase the scalibility of the server. If a file in the global "path" is editted (And it is not a log file) then the file will be added to the queue for sharing. As the server doles out files, it pops the queue... shortening the size of the queue.

  • Conclusion

    This program has taught me how easy it is to create a scalable nodejs program. Error handling is rather simple.
    I also learned how this program will be inefficient for power, using a psuedo spinlock.
